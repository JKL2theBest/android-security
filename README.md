# Лабораторная работа №3: Безопасность Android-приложений

Проект мобильного приложения на Kotlin, разработанный в рамках изучения методов анализа APK-файлов, поиска URL, эндпоинтов и скрытых секретов.

## 📱 Функционал приложения
Приложение состоит из трех основных экранов, реализованных на **Jetpack Compose**:
1.  **Профиль:** Вывод ФИО студента и номера группы.
2.  **API Данные:** Загрузка и отображение списка постов из публичного REST API ([JSONPlaceholder](https://jsonplaceholder.typicode.com/)) с использованием **Retrofit**.
3.  **Обратная связь:** Форма ввода сообщения с имитацией отправки.

## 🛠 Технологический стек
-   **Язык:** Kotlin
-   **UI:** Jetpack Compose
-   **Navigation:** Compose Navigation (для переключения между экранами)
-   **Network:** Retrofit 2 + Gson (для работы с REST API)
-   **Architecture:** ViewModel + LiveData/State (реактивное обновление UI)

## 🔍 Подготовка к анализу безопасности
В коде намеренно оставлены элементы для поиска через инструменты статического анализа (например, `APKLeaks`, `MobSF` или `apktool`):
-   **Base URL:** Прямая ссылка на API.
-   **Endpoints:** Пути запросов внутри интерфейса `ApiService`.
-   **Скрытые данные:** Возможность добавления API-ключей для тестирования сканеров секретов.

## 📦 Сборка APK
Для выполнения задания необходимо собрать две версии приложения:

### 1. Версия без обфускации (Debug)
Сборка содержит отладочную информацию и исходные названия классов/переменных.
-   **Команда:** `Build` -> `Build APK(s)`
-   **Путь:** `app/build/outputs/apk/debug/app-debug.apk`

### 2. Версия с обфускацией (Release + R8)
Код обфусцирован с помощью R8 (замена ProGuard), что затрудняет реверс-инжиниринг.
-   **Настройка:** В `build.gradle.kts` параметр `isMinifyEnabled = true`.
-   **Команда:** `Build` -> `Generate Signed Bundle / APK` -> `APK` -> `release`
-   **Путь:** `app/release/app-release.apk`
